# /figure <N> — Step 9: author + render a chapter's load-bearing figure(s)

Figures follow `{{FIGURE_POLICY}}` — **load-bearing only**. Each chapter has a **per-chapter image budget** fixed at draft time (GUIDELINES §8): the budget's designed conceptual diagrams plus any captured {{SUBJECT_SHORT}}-native screenshots, sized to the chapter's class. **ZERO figures is the EXCEPTION**, reserved for short or pure-narrative/pure-API chapters. You are the conductor for **pipeline step 9**: realize the chapter's already-fixed figure plan for chapter key `$ARGUMENTS` by **authoring each designed diagram as HTML and rendering it to a cropped PNG**. Run the `figure-designer` agent per diagram in the plan.

> **Designed diagrams are authored as HTML and rendered — NEVER generated by an image model.** The durable deliverables are the **HTML source of truth**, a **`.sources.md` trace sidecar**, and the **rendered PNG** — never an image prompt, never `NN_slug_figure.md`.

> **Inputs (read in full):** the chapter draft `03-drafts/NN_slug/NN_slug_v*.md` (its **figure plan**, fixed at Step 4), its dossier, `00-strategy/templates/FIGURE-GUIDE-{{SUBJECT_SHORT}}.md` (the house figure style), and `05-figures/_assets/figures.css` (the shared palette/components). Run after the scorecard (step 8) clears, so each figure depicts the chapter's settled core idea. Captured screenshots are kept separately; this command produces the **designed diagrams**.

## What to do — for each designed diagram in the chapter's figure plan

### Step 9a — author the HTML + sources sidecar
1. Identify the diagram's **single core idea** — the one mechanism a reader should grasp at a glance.
2. Run `figure-designer` to write `05-figures/NN_slug/figNN_x.html` — a self-contained HTML file that:
   - **Links the shared stylesheet:** `<link rel="stylesheet" href="../_assets/figures.css">` — palette and components come from there; keep figure-local CSS to layout only.
   - Wraps the diagram in the element the renderer crops to — **`id="figure"`** (the default `render.mjs` selector).
   - Holds the house style from `FIGURE-GUIDE-{{SUBJECT_SHORT}}.md`:
     - One idea, one image — graspable at a glance, technically accurate to *this* chapter.
     - Labels live **in the HTML/SVG markup** (they are real text, translation- and review-friendly) — not baked into a raster.
     - **Neutral imagery** — no vendor logos unless the topic itself requires one ({{NEUTRALITY_STANCE}}).
     - Consistent palette/treatment across all chapters via `figures.css`.
3. Write the trace sidecar `05-figures/NN_slug/figNN_x.sources.md`: a **per-label source-trace table** mapping every label in the diagram to its source path in {{AUTHORITY_SOURCE}} at {{AUTHORITY_PIN}} (per `00-strategy/SOURCE-PIN.md`). The figure asserts nothing the prose has not already traced; end with the source-trace status line. (See the reference instance's worked figure for the reference shape.)

### Step 9b — render the PNG
4. Render the HTML to a cropped, print-DPI PNG:
   ```
   node 05-figures/_assets/render.mjs 05-figures/NN_slug/figNN_x.html 05-figures/NN_slug/figNN_x.png
   ```
   `render.mjs` drives the installed Chrome via `puppeteer-core` and screenshots **only the `#figure` bounding box** at 3× scale — a tight crop, no page chrome. The **HTML is the source of truth; the PNG is the rendered artifact.** Both are committed.
5. **Deliverable per diagram = `figNN_x.html` + `figNN_x.sources.md` + `figNN_x.png`.** All three are reviewed with the chapter at **Step 12**.

## Guardrails
- A figure depicts **only what the chapter actually verified** — no invented architecture, no components absent from the dossier. Each figure cites the **same pinned source paths** as the prose it depicts (per-label trace in `.sources.md`); a label with no pinned source is fatal (HARD rule 3 — never invent {{INVENT_UNITS}}).
- Realize the chapter's **fixed figure plan** — do not invent figures beyond the per-chapter budget, and do not drop budgeted ones. If the plan seems to need a figure the chapter does not actually contain, that is an editorial signal — flag it to `09-flags/` rather than improvising.
- If a designed diagram tries to carry two ideas, the chapter is doing two things — split the figure or flag it; do not cram.

## Tooling honesty
`render.mjs` and `figures.css` exist under `05-figures/_assets/` and the reference instance has a worked reference figure. `figure-designer` is newly authored; if its spec is not yet present, author the HTML + sources sidecar as a documented manual procedure following `FIGURE-GUIDE-{{SUBJECT_SHORT}}.md` and the reference figure, and say so. Rendering requires Chrome (set `CHROME_PATH` if `render.mjs` cannot find it).

## Report
For each diagram: the core idea chosen, the three deliverable paths (`figNN_x.html`, `figNN_x.sources.md`, `figNN_x.png`), confirmation it meets the house style (one idea, real text labels, neutral, cropped via `#figure`), and that every label traces to a pinned {{AUTHORITY_PIN}} source. Note the figures are reviewed with the chapter at Step 12. Close with **"Learnings & pipeline suggestions"**; have the maintainer log it.
