JEP 444: "when it executes code inside a synchronized block or method … its carrier and the underlying OS thread are blocked for the duration." Detected via -Djdk.tracePinnedThreads / JFR event; flagged by Sonar java:S6906.

Java 21–23
synchronized pins. Mitigation: replace with ReentrantLock for any blocking call inside the lock. (Sonar java:S6906)
Java 24+ (JEP 491)
synchronized no longer pins. JEP 491: "the synchronized keyword no longer pins virtual threads." Native methods and foreign-function calls still pin.

Sources: JEP 444 (GA, Release 21) — mounting/unmounting/carrier/scheduler/pinning-on-synchronized verbatim. JEP 491 (Release 24) — synchronized no longer pins, native/FFM still do. Sonar java:S6906 — "virtual threads should not run tasks that include synchronized code." -Djdk.tracePinnedThreads / JFR pinned-thread event — JEP 444 diagnostics. Version-bound advice form: dates the JDK, never states a timeless rule.
