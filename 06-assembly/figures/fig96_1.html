Concept — prioritize by churn × pain

Debt in frozen code accrues no interest: a horrifically complex class no one has touched in three years costs nothing to leave. The same complexity in a file edited weekly costs every developer every week. Remediate where change actually happens — the hotspots — not the worst code by absolute metric. This is what makes remediation finite and defensible.

✗ Never a big-bang rewrite

It overruns, reintroduces a fresh generation of debt, and the old system keeps moving while the rewrite chases it. The playbook rejects it as the default. Attributed to Fowler's strangler-fig motivation — not a universal law.

✗ Never a baseline without a paydown plan

A baseline buys time to remediate; it is not the remediation. A debt register nobody works is theater — formalized ignoring. Pair every baseline (step 1) with the hotspot paydown (step 4) and a ratchet.
