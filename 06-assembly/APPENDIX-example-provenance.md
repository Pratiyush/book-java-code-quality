# Appendix — Example provenance: what was built versus cited

> Reader appendix. For every chapter that ships code, this table states what its **primary** companion
> example actually is, so executed fact is never confused with illustration. It answers the question a
> skeptical reader is entitled to ask: were Testcontainers, jqwik, Pact, REST-assured, ApprovalTests,
> JMH, and OpenRewrite *actually run*, or only described? The source of truth is each chapter's
> example-build record and its module on disk.

Every chapter that ships code is classified by what its **primary** companion example actually is.
**BUILT-REAL** — the named production tool/library is a real Maven dependency, built green and exercised
by a test. **STAND-IN** — an in-JVM or synthetic substitute reproduces the tool's mechanism or decision
in place of the real tool (the substitute is stated). **CITED-ONLY** — the tool is described or
configured but never built or run in the module. **RUNTIME-GATED** — the real tool is wired but needs a
runtime or network the offline build can't always provide (Docker, an NVD feed, a running provider, a
JMH measurement run, a live CI). Two Part-I/culture chapters (1 and 4) legitimately ship no module. A
design note that shapes the whole table: the discipline was to always ship an *offline-green* primary
example, so no chapter's primary example is purely RUNTIME-GATED — genuinely gated tools
(Docker/Testcontainers, the NVD scan, a Pact provider, a JMH run, OpenRewrite recipe resolution,
JFR/async-profiler) appear instead as a **secondary** status, noted in the last column. All
"green / N tests" results come from recorded local `mvn -B -Pquality verify` runs; the 45 code modules
build as one Maven reactor.
