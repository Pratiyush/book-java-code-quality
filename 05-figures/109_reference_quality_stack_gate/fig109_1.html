This is one defensible setup, not the setup. Every pick names its trade-off and an equally-valid alternative — the composition is the example; the tailored version is the deliverable. All-OSS core is possible: SonarQube Community + OSS analyzers. The full stack is code to own: build time, false-positive tuning, and config maintenance are real ongoing costs. Adopt incrementally — format first, then fast linters, then heavier analysis. Tools are necessary scaffolding; design, review, and culture are where quality is decided. Ch 1, 37 (keys 06, 84)
