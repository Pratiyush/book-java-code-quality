## Accessibility

- Alt-text: A three-lane concept map showing that every code smell (named after Fowler) comes with a refactoring that resolves it and an analyzer rule that catches it, organized into three detection modes: metric-threshold (tool-found), structural-pattern (tool- or review-found), and contract/bug-overlap (tool-found hard).
- Long-description: The figure is titled "The smell-triple: smell → refactoring → detecting rule." At the top, three pills connected by right-arrows establish the triple: "Smell (surface symptom, not a bug)" → "Refactoring (Fowler's named fix)" → "Detecting rule (tool · rule key)." Below, three horizontal sections organize smell cards by detection mode. Section A, "Metric-threshold smells — a counter vs a configurable limit — tool-found," contains six cards: Long Method (Extract Function / Decompose Conditional; PMD NcssCount method 60, Sonar java:S138, Checkstyle MethodLength), High Complexity (Extract Function / Replace Conditional with Polymorphism; PMD CyclomaticComplexity 10/80, PMD CognitiveComplexity 15, Sonar java:S3776 15), Long Parameter List (Introduce Parameter Object; PMD ExcessiveParameterList 10, Sonar java:S107 7), Large Class / God Class (Extract Class; PMD GodClass, PMD TooManyMethods 10, PMD TooManyFields 15, Sonar java:S1448), Duplicate Code (Extract Function / Pull Up Method; PMD CPD, Sonar java:S1192 threshold 3), and Data Class (Move Function; PMD DataClass). Section B, "Structural-pattern smells — AST shape match, no threshold number — tool-found or review-found," contains three cards: Feature Envy (Move Function; no reliable automated detector — review-found), Message Chains (Hide Delegate; PMD LawOfDemeter trust radius 1), and Speculative Generality (Collapse Hierarchy / Remove Delegation; no reliable automated detector — review-found). Section C, "Contract / bug-overlap smells — bytecode or flow analysis; overlaps with real defects — tool-found hard," contains three cards: Exposing Internal Representation (defensive copy or List.copyOf; SpotBugs EI_EXPOSE_REP and EI_EXPOSE_REP2; risk: caller mutation silently corrupts internal state), equals without hashCode (override both or use a record JEP 395; Error Prone EqualsHashCode, PMD OverrideBothEqualsAndHashcode), and Telescoping Constructor (Builder or record; PMD ExcessiveParameterList 10). Below the cards, three mode-key cells summarize each detection mode. A callout note at the bottom reads: "The smell is a hint to investigate, never a verdict to obey." The left border weight distinguishes mode A (thick solid), B (thin solid), and C (accent-bar orange).
