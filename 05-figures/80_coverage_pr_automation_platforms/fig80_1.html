Fig 80.1 — Diff-scoping: one discipline across coverage, the platform check, and PR feedback

Focus everything on the code this change touches. Whole-repo scope fails identically in all three facets — and a failed gate gets disabled. Scope to the diff and the gate becomes fair, adoptable, and actionable.

Whole-repo scope → the gate gets disabled
Unreachable, a stale verdict, muted noise — all three end the same way: a gate developers route around. A gate the team routes around is a net negative.
Diff scope → fair · adoptable · actionable
Fair (judges a developer only on what they wrote), adoptable (works on legacy from day one), actionable (the finding is on the line under review).
Whole-repo scope — dashed; the scope that fails
Diff scope — accent bar; the scope that works
A measure that becomes a target stops being a good measure (Goodhart). Coverage is a floor, not a goal — gating on the number alone incentivizes bad tests; reach for mutation and human review for quality, never a higher percentage. Bots are not review (they catch the mechanical, not whether it is the right change), and a green check is not good code. The gate is scoped to the diff so it is fair, delivered to the developer so it is actionable, and understood as a floor so it is not gamed — human judgment (Ch 84) is where quality actually lives.
