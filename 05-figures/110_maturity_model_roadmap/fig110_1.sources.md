## Accessibility
- Alt-text: A five-stage Java quality adoption roadmap — Foundations, Gate the basics, Deepen, Govern and observe, Sustain and evolve — entered where the pain is. (123 chars)
- Long-description: Five adoption stages run left to right, joined by right-pointing arrows, each card naming the stage, when to take it on, and the practices it carries with their chapter references. Stage 0, "Foundations" (Monday; marked as the default starting point with a left accent bar and warm header), covers VCS hygiene, a build wrapper (Ch 27), auto-format with Spotless (Ch 6), a secrets pre-commit hook (Ch 31, 35), and JUnit running in CI (Ch 20, 33). Stage 1, "Gate the basics" (on new code), covers Checkstyle/PMD plus Error Prone (Ch 16, 18), new-code coverage (Ch 23, 34), a PR gate plus branch protection (Ch 33, 35), small-PR review discipline (Ch 37), and baselining the legacy (Ch 38). Stage 2, "Deepen" (when the basics hold), covers SpotBugs plus FindSecBugs (Ch 16), SCA plus SBOM (Ch 28), ArchUnit (Ch 16, 25), a SonarQube quality gate (Ch 17), mutation testing on critical code (Ch 23), and CI speed work (Ch 33). Stage 3, "Govern and observe" (measure and govern), covers a SAST plus security gate (Ch 31, 32), fitness functions (Ch 26), dashboards and trends (Ch 38), DORA metrics (Ch 38), perf-regression gates (Ch 44), and observability plus production feedback (Ch 45). Stage 4, "Sustain and evolve" (a self-sustaining program), covers custom rules encoding org standards (Ch 18), AI governance (Ch 42), continuous debt remediation (Ch 40), culture and knowledge distribution (Ch 1, 37), and the reference stack fully realized (Ch 46). A note beneath the stages frames the sequence: start where your pain is — the order is a default, not a track; a team with a security mandate starts with the security gate, a team drowning in incidents jumps to Stage 3 observability first, and a team with a specific hotspot remediates that before anything else; climb for outcomes (fewer incidents, safer delivery, a changeable codebase), not for the level; Stage 0 is marked as the default "Monday" entry but is not obligatory if a higher stage solves the most expensive problem first; and there is no "done."
- Grayscale / contrast / legibility: PASS. Grayscale-safe — the five stages share one tinted header style and are told apart by their "Stage 0–4" numerals, names, "when" labels, and left-to-right order. Stage 0 (the default start) is the only colored element: it carries an accent left-bar and a warm header that reads light in grayscale, and the note explicitly states "Stage 0 is marked as the default 'Monday' entry," so the start marker survives in black-and-white and its meaning is spelled out in text rather than left to color. All chapter references are authored selectable text with no syntax coloring. Dark and soft ink on white/light-grey tints hold sufficient contrast at print size. No standalone color legend.
