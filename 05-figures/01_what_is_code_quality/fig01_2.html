Figure 1.2 — The cruft-tax curve

Cost per new feature over time: high vs low internal quality (qualitative illustration).

Cruft tax — Fowler defines cruft as "the difference between the current code and how it would ideally be." As cruft accumulates, every future change pays extra time to understand and work around it. High internal quality keeps the curve flat. This is a qualitative illustration; axes are labelled "low / med / high", not specific minutes. The mechanism is from Fowler (see caption).

Shape and mechanism: Martin Fowler, Is High Quality Software Worth the Cost? — martinfowler.com/articles/is-quality-worth-cost.html. The ASCII sketch in §"The half you can see and the half you can't" (draft v1) is the prose source. "Cruft" defined ibid: "the difference between the current code and how it would ideally be." Axes are qualitative; no specific minute-values are depicted or implied.
