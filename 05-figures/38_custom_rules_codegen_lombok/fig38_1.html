Pick the tool whose artifact the invariant naturally lives in. A structural or textual convention (naming, token pattern) fits Checkstyle or PMD. A type-aware rule that needs resolved overloads fits Error Prone. A pattern only visible after erasure fits SpotBugs. An architectural constraint (who may depend on whom) fits ArchUnit. The choice follows the artifact — not a ranking of the tools. (Which analyzer to run for a given rule type → Ch 17.)
Adoption guidance
Ship as WARNING first — a bespoke rule has far less field-testing than a stock one.
Promote to ERROR after gathering signal.
A custom rule is code the team maintains on every tool upgrade. Baseline & ratchet → Ch 19.
