Fig 2.1 — Cyclomatic vs. Cognitive Complexity: same paths, different nesting

Two methods, identical branch count, opposite nesting structure — equal cyclomatic score, very different cognitive score.

Variant A — flat branches

Three branches, no nesting

Guard-clause style: each condition is independent and returns immediately.

Method structure (pseudocode):

processOrder(order) { if (!order.isValid()) // +1 branch return INVALID; if (order.isPriority()) // +1 branch return EXPEDITE; if (order.isBackordered()) // +1 branch return DEFER; return NORMAL; }
Cyclomatic
4
3 branches + 1
Cognitive
3
3 × (nesting 0) = 3
Flat branches each cost +1 to cognitive score — no nesting penalty. Easy to read: one condition, one exit.
Variant B — deep nesting

Three branches, nested three deep

Same logical paths as Variant A — nested inside each other instead of side-by-side.

Method structure (pseudocode):

processOrder(order) { if (order.isValid()) { // +1 branch if (order.isPriority()) { // +1 + nesting(1)=+2 if (!order.isBackordered()) { // +1 + nesting(2)=+3 return EXPEDITE; } else { return DEFER; } } else { return NORMAL; } } return INVALID; }
Cyclomatic
4
3 branches + 1
Cognitive
6
1 + 2 + 3 = 6 (nesting penalty)
Nesting increments increase with depth: branch at depth 0 = +1, depth 1 = +2, depth 2 = +3. Reader must hold the outer context while reading inner branches.
Cyclomatic complexity = 4 (both variants) Counts independent execution paths (McCabe, 1976). Approximates minimum test cases needed. Does not penalize nesting — same score for flat or deeply-nested code.
Question answered: "How many tests do I need?"
Cognitive complexity: A = 3, B = 6 (different) Assigns incremental costs for structures that break linear reading flow, with nesting multipliers (Campbell / SonarSource). Captures the extra mental load of Variant B that cyclomatic misses.
Question answered: "How hard is this to read?"
Rule (SonarQube java:S3776): gate on cognitive complexity for readability; gate on cyclomatic for test-coverage planning. Using the wrong metric misleads — Variant B passes a cyclomatic gate of 5 but is measurably harder to read. Refactor trigger: extract method, replace nested conditionals with guard clauses, or model branches with sealed types + pattern matching (Java 21, JEP 441).
