Fig 14.1 — PECS Variance Ladder

Java generics are invariant at declaration; wildcards restore controlled flexibility at the use site.

PECS
Producer-Extends  ·  Consumer-Super  —  Item 31, Effective Java 3e (Bloch, 2018).
If a parameter produces T values, use ? extends T. If it consumes T values, use ? super T.
Declaration-site · invariant

Exact type

List<String>
Operations
✔ Read as String
✔ Add String
✘ Assign to List<Object> — invariance: not a subtype (JLS §4.10.2)
Rule: List<String> is not a List<Object> — even though String extends Object. Invariance keeps the type system sound.
Use-site · covariant (producer)

? extends T

PECS — Producer-Extends
List<? extends Number>
Operations
✔ Read as Number (upper bound)
✘ Add anything (except null) — actual type unknown
✔ Accept List<Integer> or List<Double>
Parameter produces T values for the caller to read. Widens what callers can pass in — List<Integer> satisfies List<? extends Number>.
Use-site · contravariant (consumer)

? super T

PECS — Consumer-Super
List<? super Integer>
Operations
✔ Add Integer (or subtype)
✘ Read as anything more specific than Object
✔ Accept List<Number> or List<Object>
Parameter consumes T values from the caller. Widens what callers can pass in — List<Number> satisfies List<? super Integer>.
Both at once (copy idiom) static <T> void copy(List<? extends T> src, List<? super T> dst)
src produces T (extends); dst consumes T (super). Item 31.
Unbounded wildcard <?> static void printAll(List<?> list)
The element type is irrelevant — only iteration/size operations needed. JLS §4.5.1.
Hard rule — never use a bounded wildcard as a return type. A method returning List<? extends Number> forces every caller to deal with the wildcard, propagating complexity outward. Use a concrete type in return position. Sonar java:S1452 flags wildcard return types. — Item 31, Effective Java 3e.

Sources: JLS SE 21 §4.10.2 (subtyping of parameterized types / wildcard variance); Effective Java 3e Item 31 (bounded wildcards / PECS mnemonic, wildcard-return caveat); Sonar rule java:S1452 (wildcard return types). Diagram depicts call-site variance only — declaration-site variance is not a Java feature.
