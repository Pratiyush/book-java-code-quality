## Accessibility

- Alt-text: Two-track diagram: a virtual thread unmounts and frees its carrier on the normal path, but synchronized pins it (Java 21-23).
- Long-description: A diagram titled "Fig 14.1 — Virtual thread mounting, unmounting, and the pinning trap," subtitled "JEP 444 (GA, Java 21) · JEP 491 (Java 24) · Two behaviors, one version boundary." Two stacked tracks each show two lanes — an OS thread (carrier) lane above a Virtual thread VT-1 lane — progressing left to right with arrows. The top track, "Normal path — blocking inside ReentrantLock," tagged "carrier stays free," runs: the carrier runs virtual thread VT-1, then I/O blocks and VT-1 unmounts (freeing the carrier), then the carrier picks up and runs virtual thread VT-2, then I/O is done and VT-1 remounts; in the VT-1 lane below, VT-1 is running a blocking call, then parked off the carrier, then still parked, then resumes on the carrier. An annotation quotes JEP 444: when a virtual thread blocks on blocking I/O the Java runtime automatically unmounts it from its carrier, and the carrier is reused for VT-2. The bottom track, "Pinning path — blocking inside synchronized," tagged "carrier blocked (Java 21-23)" and "fixed in Java 24 (JEP 491)," runs: the carrier runs VT-1, then is PINNED — VT-1 cannot unmount and the carrier OS thread is blocked, then is still blocked (pool starved); in the VT-1 lane, VT-1 enters synchronized, then makes a blocking call inside synchronized and cannot unmount, then is waiting with the carrier held. An annotation quotes JEP 444 that inside a synchronized block the carrier and underlying OS thread are blocked for the duration, detected via -Djdk.tracePinnedThreads / a JFR event and flagged by Sonar java:S6906. Below the tracks, a two-cell version summary: "Java 21-23 — synchronized pins; mitigation: replace with ReentrantLock for any blocking call inside the lock (Sonar java:S6906)"; and "Java 24+ (JEP 491) — synchronized no longer pins virtual threads; native methods and foreign-function calls still pin." A source line closes the figure.
