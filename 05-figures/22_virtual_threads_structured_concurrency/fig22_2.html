Fig 22.2 — Three-layer concurrency verification stack

Static detection (build time) + stress sampling (verify time) + deterministic reproduction (regression time) — complementary, not substitutes

Layer 1 — Static Analysis
Build time
Error Prone Source AST

@GuardedBy("lock") — machine-readable claim on which lock guards each field. Checker enforces at every access. Severity: ON_BY_DEFAULT ERROR.

build-failing check on annotated fields; lock-discipline at every access site

races on un-annotated fields; non-lock visibility races

SpotBugs MT_CORRECTNESS Bytecode / Heuristic

Recognizes dangerous shapes: IS2_INCONSISTENT_SYNC, DC_DOUBLECHECK, VO_VOLATILE_INCREMENT, AT_OPERATION_SEQUENCE_ON_CONCURRENT_ABSTRACTION. No annotation required.

annotation-free catch of known anti-patterns from bytecode

heuristic FPs/FNs; docs note "inaccuracy"

Checker Framework Lock Checker Pluggable Types / Sound

Full lock type system: @GuardedBy, @Holding, @EnsuresLockHeld, @GuardSatisfied. Sound: if it type-checks without error, no data races on typed fields.

a guarantee for the checked lock-race property

annotation effort; scoped to lock races only

Layer 2 — Stress Sampling
Verify time (separate build target)
JCStress OpenJDK / Probabilistic

Declares shared state in @State; racing actions as @Actor methods (order deliberately unspecified); observed values in @Result; each outcome graded with @Outcome(expect=…):

ACCEPTABLE — legal outcome
ACCEPTABLE_INTERESTING — legal reordering worth surfacing
FORBIDDEN — must never appear; if observed, defect confirmed
UNKNOWN — un-graded (test-completeness gap)

Run with -m quick for a reduced pipeline pass. GAV: org.openjdk.jcstress:jcstress-core:0.16

can prove a bug exists — observe FORBIDDEN = real defect

probabilistic; green run ≠ proof of absence; hardware-dependent; slow

"experimental" per its own README. Not a merge-blocking CI gate. Reserve for genuinely subtle lock-free / publication code.

Layer 3 — Deterministic Reproduction
Regression time (once bug understood)
java.util.concurrent primitives Forces one interleaving

CountDownLatch — release all threads at the same instant; maximize overlap
CyclicBarrier / Phaser — rendezvous at the exact check-then-act window
Single-thread executor — pin a specific ordering
Clock.fixed — make time-dependent logic deterministic

forces one interleaving; bug becomes a repeatable regression test

proves only the engineered schedule; says nothing about other interleavings

Best as:

Regression lock once a bug is found and understood via stress sampling. Pair with Layer 2.

Alternatives: jqwik / Lincheck Other methods

jqwik — property-based; checks operation sequences against a sequential spec; linearizability-style.
Lincheck — model-checks / enumerates interleavings (vs. sampling real hardware). Different method, different trade-offs.

The spec property all three approximate: JLS §17.4.5 "correctly synchronized" / data-race-free. This property is undecidable in general — each layer checks a decidable proxy. Static checks lock discipline; stress samples real interleavings; deterministic forces one. Layers are complementary: static at build catches a different subset than runtime sampling.
What none of them catches: arbitrary races outside annotated/shaped patterns; deadlock (structural); liveness (scheduler-dependent). A green scan at any layer is necessary but not sufficient. The three layers together raise confidence; they do not prove race-freedom.
Anti-pattern to retire: Thread.sleep(100)-timed "concurrency tests" — neither force a specific interleaving nor sample the space. Replace with latches/barriers (Layer 3, deterministic) or JCStress (Layer 2, stress). Flaky and slow; retire them.
