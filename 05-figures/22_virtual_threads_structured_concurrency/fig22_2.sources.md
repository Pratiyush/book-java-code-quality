## Accessibility

- Alt-text: Three-layer concurrency verification stack: static analysis (build), stress sampling (verify), deterministic repro (regression).
- Long-description: A diagram titled "Fig 14.2 — Three-layer concurrency verification stack," subtitled that static detection (build time), stress sampling (verify time), and deterministic reproduction (regression time) are complementary, not substitutes. Three columns sit side by side, each a layer with a header band naming the layer and its time, then tool cards; every card states what it "Gives" and what it "Cannot" do. Column one, "Layer 1 — Static Analysis, Build time," holds three stacked cards: Error Prone (Source AST) — @GuardedBy("lock") is a machine-readable claim of which lock guards each field, enforced at every access, severity ON_BY_DEFAULT ERROR; gives a build-failing check on annotated fields, cannot catch races on un-annotated fields or non-lock visibility races. SpotBugs MT_CORRECTNESS (Bytecode / Heuristic) — recognizes dangerous shapes IS2_INCONSISTENT_SYNC, DC_DOUBLECHECK, VO_VOLATILE_INCREMENT, AT_OPERATION_SEQUENCE_ON_CONCURRENT_ABSTRACTION with no annotation required; gives annotation-free catch of known anti-patterns from bytecode, cannot avoid heuristic false positives/negatives (its docs note "inaccuracy"). Checker Framework Lock Checker (Pluggable Types / Sound) — a full lock type system with @GuardedBy, @Holding, @EnsuresLockHeld, @GuardSatisfied, sound so that if it type-checks without error there are no data races on typed fields; gives a guarantee for the checked lock-race property, cannot avoid annotation effort and is scoped to lock races only. Column two, "Layer 2 — Stress Sampling, Verify time (separate build target)," holds one card: JCStress (OpenJDK / Probabilistic) — shared state declared in @State, racing actions as @Actor methods with order deliberately unspecified, observed values in @Result, each outcome graded with @Outcome(expect=…); the grades are ACCEPTABLE (legal outcome), ACCEPTABLE_INTERESTING (legal reordering worth surfacing), FORBIDDEN (must never appear; if observed, defect confirmed), and UNKNOWN (un-graded, a test-completeness gap); run with -m quick for a reduced pipeline pass, GAV org.openjdk.jcstress:jcstress-core:0.16; gives the ability to prove a bug exists (observe FORBIDDEN equals a real defect), cannot prove absence — it is probabilistic, hardware-dependent, and slow, "experimental" per its own README and not a merge-blocking CI gate. Column three, "Layer 3 — Deterministic Reproduction, Regression time (once bug understood)," holds one card: java.util.concurrent primitives (Forces one interleaving) — CountDownLatch releases all threads at the same instant to maximize overlap, CyclicBarrier / Phaser rendezvous at the exact check-then-act window, a single-thread executor pins a specific ordering, and Clock.fixed makes time-dependent logic deterministic; gives one forced interleaving so the bug becomes a repeatable regression test, cannot say anything about other interleavings; best as a regression lock once a bug is found via Layer 2; an alternatives note adds jqwik (property-based, checks operation sequences against a sequential spec, linearizability-style) and Lincheck (model-checks / enumerates interleavings rather than sampling real hardware). Below the columns, two notes: the spec property all three approximate is JLS §17.4.5 "correctly synchronized" / data-race-free, undecidable in general, so each layer checks a decidable proxy; and what none of them catches — arbitrary races outside annotated/shaped patterns, deadlock (structural), and liveness (scheduler-dependent) — so a green scan at any layer is necessary but not sufficient. A closing anti-pattern callout reads: retire Thread.sleep(100)-timed "concurrency tests," which neither force a specific interleaving nor sample the space; replace them with latches/barriers (Layer 3) or JCStress (Layer 2). A source line closes the figure.
