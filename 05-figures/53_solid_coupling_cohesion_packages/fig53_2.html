Figure 25.2 — The module-strength ladder

Encapsulation is not binary — climb only as far as the boundary warrants. Each rung trades ceremony for stronger guarantees.

↑
Enforcement strength
Rung / mechanism
Guarantee
Cost / ceremony
1
Package convention
names only — discipline
Agreed naming signals intent; nothing enforces it — any class can reach any other.
Zero tooling cost; zero enforcement — breaks silently under deadline pressure.
2
package-private access
Java default visibility
Compiler hides internals within a package; classes outside the package cannot reach them.
No build changes; discipline to omit public keyword. Java's most under-used encapsulation tool.
3
ArchUnit-enforced slices
slices().should().beFreeOfCycles()
A test fails on a forbidden dependency or cycle — structure is continuously checked in CI.
ArchUnit test dependency; rules authored once; catches violations before merge. Enforcement chapter's primary tool.
4
JPMS module-info
JEP 261 · compiler-enforced
Compiler forbids access across module boundaries; only explicitly exports-ed packages are visible.
module-info.java per module; dependency graph declared; reflection restrictions. Ceremony meaningful for libraries or large systems.
5
Separate build modules
Maven/Gradle multi-module
Depending on an undeclared module is physically impossible — the build fails if the dependency is not wired.
Separate build unit per module; build aggregator; physical separation. Highest ceremony — right only where the boundary truly warrants it.
By-layer (controller · service · repository)
Familiar; maps to layered architecture; easy entry point for new teams.
Risk: a feature's code scatters across packages; layer packages become low-cohesion buckets.
vs
By-feature (orders · billing)
High cohesion per feature; change stays local; natural fit for modular monoliths.
Risk: cross-feature sharing needs deliberate API packages; more up-front planning.
Rule of thumb: Use package-private by default (rung 2); add ArchUnit slices when cycle discipline matters (rung 3); reach for JPMS or build modules only when the boundary is load-bearing and team-scale warrants the ceremony. Neither by-layer nor by-feature is crowned — context decides.
