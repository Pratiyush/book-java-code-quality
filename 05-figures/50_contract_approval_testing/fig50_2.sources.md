# fig50_2.sources.md — source-trace sidecar for Fig 50.2

**Chapter:** 24 — Contract & Approval Testing (dossier key 50, folds 52)
**Figure:** Fig 50.2 — Three techniques, three questions on the service boundary and its output
**Family:** Concept-map
**One idea:** Three complementary techniques each answer a question the others cannot — contract testing asks "do the sides agree?" (no live partner), API testing asks "does the endpoint behave?" (needs live endpoint), approval testing asks "did the output change from a human-reviewed baseline?" — each is the wrong tool for the other's question, and each fails when its own reference goes wrong.
