Figure 24.1 — Pact four-stage pipeline

Consumer-driven contract guarantee requires all four stages; any half-pipeline delivers false confidence.

STAGE 1 Consumer test
  • @PactConsumerTestExt
  • @Pact method declares interaction
  • PactDslJsonBody — type-matching (stringType(), matchRegex())
  • Real client → injected MockServer
  • Writes pact JSON → target/pacts

Runs against mock, not the real provider.

→
STAGE 2 Publish to broker
  • Shared store of contracts + verification results
  • Holds consumer version selectors: deployed, released
  • Builds consumer × provider matrix

Broker is the shared authority — no broker, no can-i-deploy.

→
STAGE 3 Provider verification
  • @Provider, @PactBroker
  • @TestTemplate — one test per interaction
  • verifyInteraction() replays each request against running provider
  • @State seeds preconditions
  • Checks real response matches contract

Holds consumer's mock honest. Needs live provider.

→
STAGE 4 can-i-deploy gate
  • Queries broker matrix of consumer × provider verification results
  • Selectors: deployedOrReleased
  • record-deployment / record-release after successful deploy
  • Safe to deploy = all consuming versions verified against this provider version

Gate only meaningful with record-deployment discipline.

Half-pipeline failure mode: Stage 1 alone (consumer test against own mock) proves only the consumer's assumptions — the mock can expect data the real provider never sends. Stage 3 (provider verification) is what holds the mock honest. Run all four stages or none of the contract guarantee applies.
