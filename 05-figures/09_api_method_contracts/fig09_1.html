Fig 09.1 — The two halves of a method contract

Push as much of the contract into the type system as possible — the cheapest feedback on a broken contract is "it didn't compile"

The left-shift spectrum — move promises leftward
Comment / doc only

←←← design goal: shift leftward later / costlier feedback →
Type-carried half compile error — cheapest
Enforced by: compiler · static checker · IDE
1
Return Optional<T>, not null (Item 55) "May be absent" becomes a type fact; caller is forced to handle the empty case before touching the value. Never Optional for parameters, fields, or boxed primitives (OptionalInt/OptionalLong/OptionalDouble instead).
2
Nullness as a signature fact — JSpecify 1.0.0 @Nullable / @NonNull / @NullMarked put the null contract into the signature. JSpecify is a specification, not a checker — any conforming checker (NullAway, Checker Framework) verifies it. JSpecify 1.0.0  ·  NullAway 0.13.4
3
Minimize accessibility (Items 15–17) Every public / protected member is a promise kept forever (binary-compat burden). Make each as inaccessible as possible; prefer immutable types (smallest contract of all).
4
Real override — @Override Compiler verifies the override is genuine; catches signature drift silently. EP MissingOverride
5
Un-ignorable return values — @CheckReturnValue When the return value is the result, dropping it is a bug. @CanIgnoreReturnValue exempts builders that return this. EP CheckReturnValue  ·  SpotBugs RV_RETURN_VALUE_IGNORED  ·  Sonar java:S2201
6
No parameter reassignment — final params final makes overwriting a parameter before reading it a compile error; analyzers flag the rest. PMD AvoidReassigningParameters  ·  Sonar java:S1226
⇄
Doc / runtime-carried half runtime exception — later, costlier
Enforced by: Javadoc (states it) · fail-fast check (enforces it)
A
Fail fast on bad input (Item 49) Check parameters at the top of the method; throw immediately. Converts a wrong result far away into a clear exception at the call site. java.util.Objects vocabulary (JDK 21):
requireNonNull(T) · requireNonNull(T, String) · requireNonNull(T, Supplier<String>) — since 1.7 / 1.7 / 1.8 checkIndex(int, int) — since 9  |  checkIndex(long, long) — since 16 checkFromToIndex · checkFromIndexSize — since 9
B
Document what the type cannot carry (Item 56) @param per parameter (preconditions), @return for every non-void method, @throws per exception with its triggering condition, first sentence as standalone summary. @implSpec for the contract a subclass may rely on.
C
Defend against representation exposure (Item 50) Copy mutable objects on the way IN (constructor / setter) and on the way OUT (getter). When the stored type is already immutable, skip the copy — copies are pure overhead then. SpotBugs EI_EXPOSE_REP / EI_EXPOSE_REP2  ·  PMD MethodReturnsInternalArray / ArrayIsStoredDirectly
D
Exception conventions NullPointerException for null · IndexOutOfBoundsException for a bad index · IllegalArgumentException for other bad values · IllegalStateException for bad object state
E
Return empty, not null — collections & arrays (Item 54) A method that can return "no elements" returns an empty collection or array, never null — so callers need no null guard. (Distinct from Optional, which is for a single absent value.)
✓
Type-carried violation → compile error Caught at build time. No test run needed. Cheapest possible feedback. The compiler / static checker is the gate.
△
Doc/runtime-carried violation → runtime exception Caught at runtime — in a test, in staging, or in production. Fail-fast (top of method) moves the exception earlier; without it the failure surfaces far from the cause.
Contract across versions: every public / protected member is a promise kept across releases. Use japicmp 0.25.6 or revapi 0.15.1 in CI to compute the required semver bump (MAJOR.MINOR.PATCH). @Deprecated(forRemoval=true, since="…") signals removal with a migration path; deletion without deprecation is a binary-incompatible break (JLS ch. 13).
