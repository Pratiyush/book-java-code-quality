- Alt-text: Three-layer stack — build tools and IDEs on top, the JUnit Platform's TestEngine API in the middle, and the test engines below.

- Long-description: The figure, titled "Fig 21.1 — JUnit Platform: three-module architecture" and subtitled that one TestEngine API lets build tools run Jupiter, jqwik, and others on a single Platform (JUnit 6.1.0 current, minimum Java 17), is a three-layer vertical stack. The top layer, labelled Consumers, holds three component pills — Maven Surefire (useJUnitPlatform()), Gradle (useJUnitPlatform()), and the IDE (IntelliJ / Eclipse, built-in launcher) — with the note that they target the Platform Launcher API. A bridge between the top two layers reads "discover and launch engines via the Launcher API," pointing downward. The middle layer, labelled Platform and drawn with the heaviest border as the foundation, contains three API boxes — TestEngine API (the integration contract), Launcher API (discovery and execution), and junit-bom (version alignment) — over the description "Foundation that launches test engines on the JVM, what build tools and IDEs target directly." Four downward arrows connect the Platform to the bottom layer. The bottom layer, labelled Engines, holds four component pills: JUnit Jupiter (the programming model: @Test, @Nested, @ExtendWith), drawn with a heavier border as the focus; JUnit Vintage (deprecated in JUnit 6; runs JUnit 3/4 suites), drawn with a dashed border; jqwik (property-based; same Platform); and the Mockito extension (via @ExtendWith; hooks into Jupiter). A callout beneath the stack states the quality point: because every tool targets one TestEngine API, the whole test ecosystem — JaCoCo, PITest, jqwik, Testcontainers, Spring — composes on a single runner, so choosing JUnit is choosing the substrate the rest of Part V assumes. A legend maps the three pill styles: focus fill = JUnit Jupiter (current programming model, JUnit 5 and 6); dashed = JUnit Vintage (deprecated in JUnit 6, migration-only); plain = third-party engines on the same Platform.
