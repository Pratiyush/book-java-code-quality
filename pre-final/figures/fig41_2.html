Fig 20.2 — Coverage vs. mutation score: two independent axes of test quality

Coverage measures execution; mutation score measures fault detection. They are independent. A suite can reach 100% coverage and kill zero mutants.

X-axis — Coverage (execution)
JaCoCo six counters (verbatim): Instructions, Branches, Lines, Methods, Classes, Cyclomatic Complexity.

Measures which instructions ran while tests executed. Branch coverage (every if / switch arm) is the stronger floor.

PITest verbatim: “measures only which code is executed by your tests.”
Y-axis — Mutation score (detection)
PITest “automatically seeds” small faults, then reruns the suite per mutant.

Killed: a test failed → fault detected.
Lived: all tests passed → suite blind to that fault.

Score = killed ÷ (killed + lived) × 100%.
“It does not check that your tests are actually able to detect faults in the executed code.” PITest verbatim (pitest.org)
Coverage is necessary, not sufficient — a floor that finds untested code, never proof that tested code is well tested. Chapter thesis; folklore guard confirmed by PITest quote above.
