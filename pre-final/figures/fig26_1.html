Fig 15.1 — Static-analysis technique ladder

Four moves, each seeing more than the one below — at rising power and cost.

MOVE 1 Parse to an AST — match shapes
cost: low runs on: source
Tools: PMD (XPath / visitor on "root AST node"), Checkstyle (source tokens), Error Prone (compiler AST inside javac)
catches well
style, naming, syntactic smells, empty catch, missing break
blind spot
anything behavioral — sees shape, not runtime values
↓
+ symbols & types
MOVE 2 Resolve symbols & types
cost: low runs on: source / compiler
Tools: PMD SymbolFacade + optional TypeResolution; Error Prone (full types free inside javac)
catches well
type-aware misuse: wrong overload, raw types, LinkedList.get(int) in a loop
blind spot
flow-dependent facts — cannot tell if a value is null at a point
↓
+ control-flow graph + data-flow fixpoint
MOVE 3 Control- & data-flow analysis
cost: medium–high runs on: source / bytecode / whole-program graph
Tools: PMD "DFA visitor … building control flow graphs and data flow nodes"; SpotBugs OpcodeStackDetector on bytecode (.class); CodeQL local/global data-flow graph
catches well
null-deref, unclosed resource, lock discipline (intra); leaks/injection spanning methods (inter/global)
blind spot
intraprocedural stops at method boundary; global flow must approximate aliasing, reflection, dynamic dispatch
↓
+ source / sink / sanitizer roles
MOVE 4 Taint tracking
cost: high runs in: CI / nightly, not pre-commit
Tools: CodeQL — taint "extends data flow … steps in which data values are not necessarily preserved, but the potentially insecure object is still propagated" (e.g. y = x+1: y "derived from x"). Semgrep — AST→IL + constant-prop + taint; "No path sensitivity / No pointer or shape analysis / No soundness guarantees."
catches well
SQLi, XSS, command injection — attacker-controlled source → dangerous sink
blind spot
unmodeled sources or sanitizers; Semgrep engine is intraprocedural
power & cost
↑ more power
↓ cheaper
Move 1 runs on every
keystroke; Move 4
runs nightly in CI
Soundness ↔ Completeness
Sound — no false negatives; all real bugs reported (costs false positives)
Complete — no false positives; every report real (costs false negatives)
No analyzer is both for a non-trivial property (Rice's theorem → halting problem). Every tool picks a point on this spectrum — that choice is its character.
Checker Framework: "values soundness over limiting false positives." Semgrep OSS: "No soundness guarantees."
