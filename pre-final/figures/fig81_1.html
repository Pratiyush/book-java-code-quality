Rungs 1–3 · SPEED
Feedback — deliberately skippable
Fast checks on the developer's machine. Bypassable on purpose: git commit --no-verify skips a hook, and hooks run only where installed. So they are feedback, never the gate. Slow checks here get disabled — keep them fast.
Rungs 4–5 · TEETH
Enforcement — deliberately unbypassable
The required CI check + branch protection + merge queue re-run everything regardless. This is the gate; a hook never is.
Feedback makes enforcement painless; enforcement makes feedback safe to skip. Local↔CI parity — the build wrapper and pinned tool versions — is what lets “green locally” reliably predict “green in CI,” so the first time a developer meets the gate is at their keyboard, not in a ten-minute CI round-trip. Trunk-based development is reported by DORA as correlating with delivery performance, given a trustworthy gate, reliable tests, and feature flags — branching strategy stays context-dependent and no single model is crowned.
