Fig 105.1 — The performance-regression gate: a fitness function that knows its own noise

Measure against a baseline, compare relative (not absolute) with statistics and a tolerance band, and flag-then-investigate a small diff rather than hard-block — fail safe when the measurement cannot be trusted.

Stage 4 · Baseline Ratchet deliberately Managed, not frozen
  • A sanctioned perf change moves the baseline — like a coverage ratchetCh 34
  • Track the trend on a dashboardCh 38 — the thousand cuts become visible over time
A baseline never updated drifts out of meaning; one updated carelessly hides regressions.
Handling noise — the hard part The same code measures differently run to run (shared runners, CPU frequency scaling, neighbouring jobs, JIT/GC scheduling). A naive absolute-threshold gate flaps on this — and a flapping gate gets disabled.
Relative, not absoluteCompare against a moving baseline — robust to environment drift.
Multiple runs + statisticsConfidence intervals tell a real regression from jitter.
Tolerance bandsA band wide enough to absorb noise, tight enough to catch the cut over time.
Stable environmentA dedicated perf env, off shared CI runners.
Flag-then-investigateOn small diffs — fail safe, never a noisy hard-block.
Placement: nightly, not the PR fast path Perf gates are slow (warmup + multiple runs) — run on main, nightly, or a dedicated perf pipeline. The PR fast path is reserved for cheap, stable microbenchmarks of critical code.Ch 33
Green means stable, not fast A green gate means "no regression versus baseline" — not "fast enough." If the baseline itself does not meet a real target, the gate is protecting an inadequate status quo.Ch 43
Complements production monitoring The gate catches regressions pre-release; monitoring catches what slips past it. Neither alone is complete.Ch 45
Why a perf gate is a different kind of gate. A failing unit test is signal — the assertion is deterministic, so hard-blocking is correct. A "failing" perf measurement might be noise, so the gate must distinguish a real regression from jitter and fail safe (flag, not block) when it cannot be sure. The meaningful gate is at the macro / load level — gating on a microbenchmark alone can block a harmless change or pass one that regresses the system while the benchmark stays green. Performance, gated honestly, is a fitness functionCh 26 that knows its own noise and its own limits.
