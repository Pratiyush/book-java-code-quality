Three instruments of Java immutability

Each instrument closes a specific gap — composed, they satisfy Effective Java Item 17 Rule 5

Instrument
Guarantee provided
Gap it leaves
Records JEP 395 · GA Java 16

Provides

Private final fields per component; compiler-derived canonical constructor, accessors, equals, hashCode, toString; implicitly final class

Gap

Makes the component reference final — does not copy or freeze a mutable component (shallow only)

↓
Immutable collections List/Set/Map.of · copyOf · JDK 9+

Provides

Throws UnsupportedOperationException on any structural mutation; copyOf produces a snapshot no caller can reach through

Gap

Freezes the collection, not its elements; unmodifiableList(x) is a view — still reflects mutation of the source x

↓
Defensive copies Effective Java Item 50 · compact constructor

Provides

A private snapshot no caller can reach; copy before validate; List.copyOf(items) in the compact constructor seals Rule 5

Cost / limit

Allocation per construction; avoid clone on subclassable types; not needed if the component is already immutable

Using the feature ≠ getting the guarantee. A record with a List<LineItem> field satisfies Rules 1–4 (Item 17) automatically. Rule 5 — exclusive access to mutable components — requires the compact constructor to call List.copyOf(items). Without it, the reference is final; the list is not.
