Throughput and stability correlate — not a trade-off. Elite teams are both fast and stable; the same practices (small batches, automated gates, fast feedback) enable both.

Gameable & incomplete: split deploys juice frequency; architecture, debt reduction, and mentoring generate no DORA signal. Metrics measure the system, never individuals. Use as questions, not verdicts.
Use — productivity (multi-axis)

SPACE framework (Forsgren et al. 2021)

S
Satisfaction & well-being
Developer fulfillment; sustainable pace
P
Performance
Outcomes and quality of work delivered
A
Activity
Volume of actions (commits, PRs, deploys)
C
Communication & collaboration
Knowledge sharing; unblocking teammates
E
Efficiency & flow
Uninterrupted work; low friction; fast feedback
Rule: use 2–3 dimensions together — never Activity alone. Activity is exactly the LOC / commit-count trap: it measures output, not value.
Limitation: no single axis can be gamed because the others expose it; observational / survey-based — needs honest self-report. Never per-individual ranking.
Refuse — vanity outputs

Gameable. Value-blind. Morale-harming.

✕
Lines of code (LOC)
Pad code to score; removing lines is invisible
✕
Commit count
Split commits to inflate; measures activity not value
✕
PR count
Pressure to merge; discourages large safe changes
✕
Velocity / story points as productivity
Estimates inflate; points become the goal, not delivery
✕
Individual leaderboards
Architecture, mentoring, debt — generate no score; strongest engineer ranks last
Goodhart's law: when a measure becomes a target, it ceases to be a good measure. Every vanity metric triggers exactly this corruption. The hook's LOC ranking — padded code, no mentoring, strongest engineer ranked last — is Goodhart made flesh.
The unifying rule (applied at team / system level, never individual): measure outcomes not outputs · trends not absolutes · counter-metric everything · treat every number as a question, not a verdict. Quality trends that complement DORA/SPACE: defect-escape rate · MTTR · flaky-test rate (Ch 20) · debt-ratio trend (key 59) · new-code coverage trend (Ch 34) — each as a trend with a counter-metric, never a standalone target.
