# A note on how this book was made — AI authorship and disclosure

This book is **AI-produced**. Its prose, its diagrams, and its worked code examples were generated by
an AI system working inside a defined production pipeline, under human direction and review. This page
states that plainly and describes how the work was controlled, so a reader knows what they are holding
and what they can rely on. It is a factual account of the process, not legal advice and not a marketing
claim.

The disclosure is made for two reasons. The first is honesty: a reader deserves to know when a machine
wrote the words. The second is obligation — storefront AI-content policies and transparency rules such
as the EU AI Act ask a publisher to state, factually, that content is AI-generated and how it was
produced. Both are served by describing the process truthfully rather than hiding it.

---

## What was AI-generated

- **The chapter text.** Every chapter was drafted by a large language model — **Claude (Anthropic's
  Opus model)** — from a research dossier prepared for that chapter. No chapter was written by a human
  author; the human role is direction and review, described below.
- **The diagrams.** Figures in this book are designed diagrams, authored as HTML and rendered to an
  image. They are not produced by an image-generation model. Where a screenshot appears, it shows a tool
  running against this book's own example code, not a third party's interface.
- **The worked examples.** The runnable code examples were generated for this book. They are original
  to it — written to make a point, not copied from another project — and each displayed snippet is a
  region taken from a file that compiles.

## How the writing was controlled

The pipeline that produced this book applies a set of checks to every chapter. The following describe
how the work is controlled; where a check is part of the design but has not yet completed for this
edition, that is stated openly in *"What is and is not complete"* below.

- **Every fact traces to a pinned source.** Each version number, configuration key, rule identifier,
  tool flag, API signature, dependency coordinate, benchmark figure, and quoted claim is traced to a
  fixed set of authoritative sources — the project's pinned authority set, frozen at a recorded date
  rather than read from a moving "latest" page. The runtime baseline is Java 21 (LTS) with Java 25 (LTS)
  called out where a later change matters. A detail that could not be traced to that pinned set was cut
  or marked unverified, never invented.
- **A neutral survey, with honest limitations.** This book surveys tools and practices; it does not
  crown a winner. Each option is given its strongest case **and** its hardest limitation, including when
  not to reach for it. Comparative claims carry a cited source.
- **Independent, different-model review.** The pipeline is designed so that the checks which judge a
  chapter — an originality scan against the wider web, an adversarial review, and the quality scorecard —
  run on a **different** AI model from the one that wrote the chapter. This is deliberate: a model is a
  poor judge of its own output, so the reviewing model is not the drafting model.
- **Runnable examples are built.** Each chapter's companion code is compiled and checked by the build,
  with static-analysis checks run over it, so the examples in the book are backed by code that actually
  runs rather than code that merely looks plausible.
- **Human review at two gates.** Two decisions in this pipeline are reserved for a human and cannot be
  made by the AI: **per-chapter approval**, where a person signs off that a chapter is fit to ship, and
  the **whole-book release gate**, where a person signs off on the assembled manuscript. The selection of
  what this book covers was also confirmed by a human.

## What you can rely on

- The technical facts are **source-traced**, not recalled from a model's memory. Each is tied to a
  recorded source at a fixed version.
- The book states **limitations** as well as strengths. No tool or practice is presented as free of
  cost or risk.
- The examples are **backed by code that compiles**.
- Where something could not be verified, the pipeline's rule is to cut it or flag it rather than
  present it as fact. AI systems can still be wrong, and this disclosure does not claim the book is
  error-free — it describes the controls applied to reduce error and the channel for reporting any that
  remain.

## What is and is not complete

This page reflects the production state of this edition honestly.

- **Drafting is complete** for every chapter, on the model named above.
- **Companion code is built and checked** across the book.
- **Independent review is partial.** The independent, different-model checks have been completed for
  some chapters and not yet for others; the independent originality and adversarial reviews are part of
  the design and had not been completed for this edition at the time of writing.
- **Human approval is the release gate.** No chapter ships, and the book is not released, until the
  human approval gates above are passed. This edition is disclosed as machine-authored and human-gated;
  the internal record of which model produced each chapter and which checks have cleared is kept in the
  project's per-chapter provenance log.
